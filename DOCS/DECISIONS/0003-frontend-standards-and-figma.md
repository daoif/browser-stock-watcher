# 决策：前端规范与 Figma

## 结论
- 前端默认使用 Vue 3 + Tailwind CSS。
- 不使用 UI 组件库。
- Figma MCP 作为前端规范工具写入项目约束，用在页面结构、信息架构和关键交互的设计校对上。

## 背景
- 用户明确给出了偏好的技术栈和前端风格：Node.js、Vue 3、Tailwind、无 UI 框架。
- 用户明确反对边框套边框、间距套间距、为了局部组件规范而牺牲整体可用空间。
- 当前环境已经安装并登录 Figma，可以把它作为前端约束工具，而不是可有可无的附属品。

## 取舍
- 选择了什么：纯白简洁、结构性边框、整体优先、短操作路径、Figma 先梳理结构再编码。
- 放弃了什么：依赖 UI 组件库快速拼装页面；每个组件都加边框、外边距和内边距的堆叠式做法。
