# 计划：浏览器库存监听扩展成品交付

## 目标
- 一次性做出一个可安装、可配置、可真实使用的浏览器扩展。
- 用户安装后，可以配置多个目标地址；浏览器启动后和定时运行时都能检查目标状态；命中成功规则后收到系统通知。
- 全流程不依赖 mock，不用假数据，不留关键细节到后续拉扯。
- 仓库首发前把 `.gitignore`、README、CHANGELOG、release notes、GitHub Actions 和本地 git 仓库初始化全部补齐。

## 设计蓝图
- 扩展采用 Manifest V3。
- 点击扩展图标时，直接打开扩展自己的完整标签页页面，地址是 `chrome-extension://.../options.html`；不做 popup 小窗，不做页面内悬浮提示。
- 后台使用 `runtime.onStartup`、`alarms`、`tabs`、`storage`、`notifications`。
- 每个目标配置至少包含：名称、检查地址、成功 URL 正则、缺货关键词列表、检查间隔、启停状态。
- 检查主路径使用后台非激活标签页真实导航；不把后台 `fetch()` 作为主实现。
- 命中成功规则后发浏览器标准通知；通知进入系统通知区。
- 遇到 Cloudflare 挑战时，先等待一段时间判断是否自动放行；未放行则记录挑战状态并退避，不误报可购买。
- 发布时只维护中文版文档；GitHub 仓库首页放 QQ 交流群入口。
