## 项目约定（项目特有）

### 项目目标
- 这是一个浏览器扩展项目，用来监听多个可配置目标地址的状态变化。
- 扩展只做检查和通知，不做自动购买、自动提交、自动点击下单。
- 扩展点击图标后直接打开设置页；通知使用浏览器标准通知并投递到系统通知区。

### 交付标准
- 默认交付成品，不接受 MVP、mock、占位实现。
- 任务只有在“用户拿到手后能直接合理使用”时才算完成。
- 只要还存在关键路径断点、关键权限未接、关键错误未处理，就不能标完成。

### 技术栈
- Node.js 生态。
- 前端使用 Vue 3。
- 不使用 UI 组件库，使用 Tailwind CSS。
- 浏览器扩展以 Chrome / Edge Manifest V3 为基线。

### 前端规范
- 先看整体，再定组件，不按零件堆样式。
- 页面整体保持纯白、简洁、克制。
- 边框只用于结构性分隔，不给每个卡片、列表项、按钮都套边框。
- 间距只归属于一层，不做 padding、margin、padding 叠加。
- 留白、边框、密度优先考虑整体可用空间，不为了“组件规范看起来完整”而牺牲实际使用空间。
- 页面结构、信息架构、关键交互有明显变化时，优先把 Figma MCP 当成前端规范工具使用，先整理结构和交互，再编码。

### 文档要求
- 根目录维护 `DOCS/README.md`、`DOCS/SUMMARY.md`、`DOCS/OVERVIEW.md`、`DOCS/STATUS.md`。
- 新增或移动 `DOCS/` 文档时，必须同步更新 `DOCS/SUMMARY.md`。
- 当前目标、下一步、风险变化时，必须同步更新 `DOCS/STATUS.md`。

### 风险清单
- 目标站点可能存在 Cloudflare 挑战，检查逻辑必须优先使用真实后台标签页导航，而不是后台直接 `fetch()`。
- 任何“为了先跑通而写的假逻辑”都不允许进入主分支。
